# 10 分钟视频录制脚本蓝图

## 视频定位

这不是一个单点模型演示器，而是一个面向政府治理与金融政策研判场景的工程化智能推演平台。视频需要突出四件事：

- 能把自然语言政策转成结构化冲击
- 能驱动多智能体与市场仿真
- 能用历史窗口验证可信度
- 能输出监管优化建议和正式报告

建议采用 `纯录屏 + 配音`，不安排真人出镜。全片以系统界面为主，字幕只做时间段提示和核心结论强化。

## 固定录制动线

`系统总览 -> 政策实验 -> 历史验证 -> 研判分析（行为诊断） -> 研判分析（监管优化） -> 结果归档 -> 回到系统总览`

录制原则：

- 少切页，多纵深，每个页面只回答一个核心问题
- 每次切页后停 1-2 秒，方便字幕和配音对齐
- 重要图表优先停留在主图、KPI 卡片、政策传导链和历史对照图，不长时间滚动表格

## 分段脚本

### 0:00 - 0:40 开场

画面停留在 `系统总览` 首屏。

配音建议：

> 在政府治理和金融政策研判场景中，一个核心问题是，政策发布之后会如何传导、如何影响市场、又如何验证推演是否可信。数治观澜提供的不是单点模型问答，而是一套从政策理解、多智能体推演、历史验证到策略建议与结果归档的完整工程闭环。

### 0:40 - 1:30 系统总览

操作动线：

- 展示平台定位卡片
- 展示系统工作流
- 展示当前可运行模块

配音重点：

- 人工智能不仅用于生成解释
- 还参与政策理解、行为驱动和决策支持
- 平台具备可复盘、可追溯、可归档的工程特性

### 1:30 - 4:00 政策实验

建议场景：

- 优先选择印花税下调 + 流动性支持类模板

操作动线：

- 进入 `政策实验`
- 选择模板并展示政策文本
- 启动会话仿真
- 自动运行几步
- 展示市场路径、KPI、传导卡片和反事实 A/B 对照

配音重点：

- 政策如何被编译成结构化冲击
- 多智能体如何形成不同交易意图
- 市场撮合如何反映到价格、成交量和风险热度变化上

### 4:00 - 6:00 历史验证

操作动线：

- 进入 `历史验证`
- 展示系统自动抽取历史窗口政策与重大新闻
- 展示被测试政策文本
- 展示仿真走势与真实走势同图对照
- 展示综合拟真评分和偏差解释

配音重点：

- 平台不是只会生成一条漂亮曲线
- 而是能在真实历史窗口下检验政策逻辑一致性
- 这部分体现的是工程可信度和实验验证能力

### 6:00 - 7:20 行为与风险诊断

操作动线：

- 进入 `研判分析`
- 打开行为与风险诊断
- 展示 CSAD、PGR/PLR、波动聚集、传播链或干预前后对照

配音重点：

- 平台不仅输出价格路径
- 还能解释群体行为和风险扩散机制
- 这让 AI 推演结果具备更强的可解释性

### 7:20 - 8:40 监管优化

操作动线：

- 切到监管优化标签页
- 展示候选动作、A/B 差分、Pareto 前沿和推荐方案

配音重点：

- 项目不止于“看结果”
- 还能给出策略建议
- 这体现了 AI 在决策支持中的工程价值

### 8:40 - 9:30 结果沉淀

操作动线：

- 展示实验报告预览
- 点击导出实验归档
- 展示归档文件列表

配音重点：

- 结果可以沉淀为报告、摘要和归档材料
- 平台具备汇报、复盘和留痕能力

### 9:30 - 10:00 结尾总结

画面回到 `系统总览`。

收束句建议：

> 数治观澜完成了从政策理解、多智能体推演、历史验证到策略建议与结果输出的闭环，是一个面向政府与金融治理场景的工程化 AI 智能推演平台。

## 彩排检查

- 页面切换是否顺畅
- 配音是否和操作同步
- 每个模块是否都回答了一个明确问题
- 全片是否足够体现工程性，而不是单纯展示性

## 录制时应避免

- 不建议首次录制时依赖外部 API，以免网络不稳定影响画面节奏
- 不建议在视频中长时间停留日志、原始 JSON 或测试输出
- 不建议把同一页的所有图表都扫一遍，优先保留最能说明问题的主图
