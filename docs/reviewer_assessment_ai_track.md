# 评委视角审阅报告（人工智能应用赛道）

## 总评

项目具备“政策输入 -> 多智能体决策 -> 市场撮合 -> 行为诊断 -> 材料导出”的完整闭环，属于典型 AI 应用工程化作品。与纯模型问答类作品相比，工程深度与可解释性更好，适合答辩展示。

## 优势

1. 闭环完整：不仅有模型调用，还有仿真执行、指标诊断、报告导出。
2. 稳定性设计到位：支持离线演示，且现已支持在线优先失败自动回退。
3. 可解释性强：有行为金融指标、传导链、A/B 对照与反事实分析。
4. 交付友好：具备一键检查与材料导出路径，便于评审复核。

## 硬伤与扣分风险

### 必须改（P0）

1. 运行依赖必须闭环：锁定依赖中曾缺导出组件会导致主流程导入失败。
2. 自检必须可信：脚本不能出现“测试失败却显示 PASS”的误导。
3. 答辩口径一致：文档中不能出现固定测试通过数字，需引用最新报告。

### 建议改（P1）

1. 增强“数据飞轮”展示：把数据来源、抽取事件、因子摘要前置到高级分析页。
2. 强化“反事实”叙事：将推荐干预时机和 A/B 差异用可视化直接呈现。
3. 统一中英文术语：界面、脚本、文档中的术语保持一致，减少评委理解成本。

### 锦上添花（P2）

1. 增加评审专用只读模式（隐藏调试信息）。
2. 输出“本次演示运行证据”卡（在线成功率、回退次数、最近错误类型）。
3. 增加提交包自动命名脚本，降低材料格式错误风险。

## 推荐答辩话术（30秒）

本项目不是“套壳大模型”，而是将大模型能力嵌入到可复现的政策仿真工程里。系统默认在线 API 优先，遇到超时或限流会自动回退离线，保证现场演示不断流。我们还能用反事实 A/B、行为金融指标和导出报告解释“为什么会得到这个结果”。
