# 冗余代码三分法清单（价值转化优先）

## 分类标准

- 保留并前端展示：能够直接支持答辩主叙事（可讲、可看、可验证）。
- 后端对齐再展示：当前未直接可视化，但有明确业务价值与测试覆盖。
- 候删：仅历史遗留/实验碎片，无法在答辩中解释价值，且无关键依赖。

## 第一批清单

| 模块 | 当前状态 | 分类 | 价值转化动作 |
| --- | --- | --- | --- |
| `data_flywheel/*` | 有测试覆盖，主流程展示弱 | 保留并前端展示 | 在“高级分析/价值接线”增加数据来源、事件主题、影响等级摘要卡 |
| `ui/policy_lab.py` 反事实函数 | 能产出 A/B 对照评分 | 保留并前端展示 | 在“高级分析/价值接线”显示推荐干预时机与价差曲线 |
| `verify_config.py` | 存在与现行策略不一致断言 | 后端对齐再展示 | 修正 SMART 校验并强制失败退出码 |
| `demo_scenarios/hybrid_replay_abuse_minimal` | 开发辅助，不完整答辩场景 | 后端对齐再展示 | 保留为研发样例，明确不进入评委主链路 |

## 候删判据（删除前必须同时满足）

1. 主流程不可达，且测试不覆盖。
2. 无法转化为答辩展示资产。
3. 删除后无接口破坏与回归风险。

## 删除流程

1. 先在此文档标记“候删”。
2. 增加替代说明（若有）。
3. 跑全量回归后再删除。
4. 在变更日志记录删除理由与影响面。
