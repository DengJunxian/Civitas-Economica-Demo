# Demo Script (评委答辩版)

## 目标

在 10 分钟内完成“问题场景 -> AI 决策 -> 市场反馈 -> 解释诊断 -> 回测验证 -> 材料导出”的闭环展示。

## 演示主路径

### 0:00 - 1:00 开场

- 展示首页五入口：答辩模式、专家模式、历史回测、行为金融诊断、系统说明。
- 用一句话定义作品：这是一个面向政策冲击与舆情扰动场景的多智能体社会经济金融仿真系统。
- 强调两点：`DEMO_MODE` 可离线演示；核心不是单次大模型问答，而是“分析 -> 决策 -> 撮合 -> 反馈 -> 解释”的闭环。

### 1:00 - 4:00 答辩模式

- 选择 `tax_cut_liquidity_boost` 或 `regulator_stabilization_intervention` 场景。
- 点击自动播放，让时间线自行推进，避免现场频繁手动操作。
- 重点讲四件事：
  - 场景输入是什么。
  - Analyst / Manager 如何形成结构化结论。
  - 市场 KPI 如何变化。
  - A/B Compare 如何体现“有无政策/监管”差异。

### 4:00 - 6:30 专家模式

- 打开专家模式中的“决策证据流”。
- 强调展示的是结构化 evidence，不是原始 chain-of-thought。
- 快速展示 LOB、传播热图、政策传导链、风险事件时间轴，说明模型输出如何影响市场结果。

### 6:30 - 8:00 行为金融诊断

- 打开行为金融诊断页，展示 CSAD、恐慌热度、羊群效应等指标。
- 把“AI 决策”和“行为金融结果解释”连起来，说明系统不是黑盒出结果。

### 8:00 - 9:00 历史回测

- 打开历史回测页，展示策略结果与稳健性。
- 强调仿真页和回测页复用同一套因子与解释框架。

### 9:00 - 10:00 收尾

- 点击“自动生成比赛材料”。
- 展示生成的 `competition_summary.md`、`design_outline.md`、`demo_script_10min.md`、`figures/index.json`。
- 用一句话收尾：系统既可离线稳定演示，也可接入真实模型/API 进入 live 模式。

## 备选演示路径

### 备选路径 A：只用答辩模式完成全程

- 如果现场网络不稳或图表切页卡顿，停留在答辩模式完成核心展示。
- 只讲场景、KPI、三段叙事、A/B Compare、材料导出，依然能讲清主价值。

### 备选路径 B：直接展示导出材料

- 如果浏览器或 Streamlit 页面异常，直接打开 `outputs/competition_materials/` 下已生成材料。
- 用静态文档补讲问题定义、技术路线、实验结果与可复现性。

## 现场应避免的演示内容

- 不建议现场以 `python main.py` 作为主入口，优先用 `streamlit run app.py` 或启动脚本。
- 不建议现场依赖外部 API 做首次展示，网络问题会掩盖作品质量。
- 不建议点击 `demo_scenarios/hybrid_replay_abuse_minimal` 作为比赛主场景，该目录不是完整答辩场景。
- 不建议在答辩中展开过长日志、测试输出或原始推理文本。
