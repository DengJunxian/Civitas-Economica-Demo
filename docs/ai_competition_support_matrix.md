# AI 赛道答辩支撑矩阵

## 1. 问题定义（评委第一问）
- 目标问题：在政策冲击和舆情扰动下，构建可解释、可复现、可演示的社会经济市场智能仿真系统。
- 传统痛点：静态可视化无法解释决策链；纯模型调用缺少工程闭环与稳定展示能力。
- 本项目回答：以多智能体 + 推理路由 + 市场微结构仿真 + 行为金融诊断形成闭环。

## 2. AI 方法链路（输入 -> 预处理 -> 推理 -> 后处理 -> 展示）
- 输入：
  - 政策/事件文本、场景配置、历史行情与仿真状态。
- 预处理：
  - 结构化场景加载、市场数据归一化、文本因子抽取。
- 推理核心：
  - `ModelRouter` 多模型优先级调用 + 超时/失败回退 + 本地缓存。
  - 多角色智能体协同（Analyst / Manager / Trader）。
- 后处理：
  - 决策结构化、风险约束映射、指标计算（CSAD、风险热度、回撤等）。
- 展示输出：
  - 答辩模式三段式叙事、A/B 对比、专家图谱、行为金融诊断与材料导出。

## 3. 自主实现边界（避免“套壳调用”质疑）
- 自主实现：
  - 场景编排与演示链路、推理路由回退策略、交易仿真调度、可视化答辩页面、材料自动生成。
- 第三方能力：
  - 模型 API（DeepSeek/智谱）、开源科学计算库、AkShare/yfinance 数据接口、Streamlit/Plotly。
- 边界表达建议：
  - 强调“本项目核心价值在 AI 工具组合与工程化落地，而非单一模型接口调用”。

## 4. 现场演示主链路（5 分钟）
- 入口：首页 -> 答辩模式。
- 步骤：
  1. 一键载入场景。
  2. 自动播放并展示 KPI 与叙事链。
  3. 切换 A/B World Compare。
  4. 打开专家模式展示传导链与风险时间轴。
  5. 打开行为金融诊断页展示报告。
- 兜底：
  - 无 API Key 可使用 DEMO_MODE 完整演示。
  - 场景列表已过滤不完整目录，避免现场点击即崩溃。
